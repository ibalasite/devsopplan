1. Executive Summary

MYDEVSOP 是一個基於 Claude Code skill 系統的 AI 驅動軟體開發 SOP 自動化工具，將完整軟體開發生命週期（SDLC）串成閉環——從最初的產品構想（BRD）到程式碼上線、GitHub PR 建立，全程 AI 驅動。

開發者只需撰寫一份 BRD（Business Requirements Document），執行一個指令（/devsop-autodev），系統自動執行 31 個 STEP，依序生成所有文件（PRD / PDD / EDD / ARCH / API / SCHEMA）、測試（BDD Gherkin / TDD 循環）、程式碼（RED→GREEN→REFACTOR）、基礎設施（Kubernetes manifests / HPA）、CI/CD pipeline（GitHub Actions）、Secrets 管理腳本（五層防護），以及完整的 HTML 文件網站，最終自動建立 GitHub PR。

每個帶有 Review 標記的 STEP 由對應領域專家型別的獨立 AI subagent 執行（Product Manager / Software Architect / Security Engineer / Database Optimizer 等），確保文件與程式碼品質達到業界標準，且每輪 Review finding 必須全數修復或明確標記 TODO[REVIEW-DEFERRED] 方可進入下一 STEP。

本工具定位為個人開發者和小型新創團隊的 AI 共同駕駛員，目標是讓「不會寫程式的人也能全程使用」，同時讓有經驗的工程師從重複性 SDLC 工作中解放，專注於架構決策與業務創新。

SDLC 文件層次定位： BRD (Why) → PRD / 本文件 (What) → PDD (How it works) → EDD (How to build)

2. Problem Statement

2.1 現狀痛點

軟體開發團隊在 SDLC 中面臨以下系統性問題：

  • 文件與程式碼漂移（Drift）：PRD 修改了但 EDD 沒更新，ARCH 改了但 API 規格沒同步，程式碼寫了但測試缺失。小型團隊往往沒有專職的 PM 或 Tech Writer。
  • Review 品質不一致：Code review 取決於 reviewer 的個人知識、精力與時間，無法標準化。沒有人能每次都完整執行 OWASP Top 10 審查。
  • 重複性工作耗時：從 BRD 生成 PRD、從 PRD 生成 EDD、從 EDD 生成 API spec——這些工作高度結構化，卻每次都需要人工完成。一份中型系統的完整文件集需要 2-4 週人工撰寫。
  • 跨職能溝通成本高：PM 寫的 User Story 工程師看不懂，工程師的架構圖 PM 無法追蹤，非技術創始人完全無法評估進度。
  • 小型團隊缺乏完整 SDLC 執行能力：一個 2-3 人的新創公司，無法同時配備 PM、架構師、QA、DevOps。

2.2 根本原因分析

  • SDLC artifact 相互依存，人工維護導致上下游文件不同步。BRD → PRD → EDD → ARCH → API → SCHEMA → BDD → TDD 是嚴格的依賴鏈，任何環節修改都需觸發下游同步。
  • Review 品質取決於個人知識與精力，無法標準化。人類 reviewer 有認知負荷上限，不能每次可靠地記起所有 OWASP 向量和 SQL 效能陷阱。
  • 文件生成高度結構化，人工執行效率低。這些工作對 AI 而言是天然適合的任務，卻佔據工程師 20-40% 的非核心時間。

2.3 機會假設

  • H-1：AI agent 可靠執行多步驟 pipeline → 小型團隊 SDLC 完整度從 40% 提升至 90% 以上。
  • H-2：領域專家型 AI subagent Review → 文件品質達到資深人類 reviewer 水準。
  • H-3：雙層輸出（技術摘要 + 白話說明）→ 跨職能溝通成本降低 50% 以上。
  • H-4：完整 31-STEP pipeline 在 60-90 分鐘內完成 → 從構想到 PR 週期縮短 80% 以上。

4. Scope

4.1 In Scope（v2.4 本版本交付）

主要入口 Skills（7 個）：

  • /devsop-autodev：全自動 31-STEP AI 開發流水線（BRD → PR）
  • /devsop-idea：構想輸入入口（模糊想法 → BRD + IDEA.md，自動銜接 autodev）
  • /devsop-change：變更管理（分析影響範圍，只重跑受影響的 STEP）
  • /devsop-repair：健康修復（診斷中斷的 pipeline，從斷點繼續）
  • /devsop-demo：3 分鐘快速體驗（URL Shortener 範例 BRD，只跑 STEP 03）
  • /devsop-project-status：專案健康儀表板
  • /devsop-update：MYDEVSOP 工具更新

生成 Skills（17 個）： gen-brd, gen-prd, gen-pdd, gen-edd, gen-arch, gen-api, gen-schema, gen-diagrams, gen-test-plan, gen-bdd, gen-client-bdd, gen-client-tdd, gen-k8s, gen-cicd, gen-secrets, gen-html, gen-readme

Review Skills（9 個）： brd-review（Product Manager）, prd-review（Product Manager）, pdd-review（UI Designer）, edd-review（Backend Architect）, arch-review（Software Architect）, api-review（API Tester）, schema-review（Database Optimizer）, code-review（Security Engineer）, test-review（Model QA Specialist）

工具 Skills（9 個）： lang-select, tdd-cycle, align-check, align-fix, align-report, rpa-test, quality-loop, product-review, config

稽核 Skills（2 個）： test-audit（7 種假測試 Pattern）, impl-audit（5 種假實作 Pattern）

平台支援： macOS / Linux（bash）+ Windows（PowerShell 5.1+ 原生）+ Skills 全平台（Claude Code AI 執行）

4.2 Out of Scope（明確排除）

  • 多使用者協作 / 團隊管理功能（原因：個人工具定位，後續有需求再設計）
  • 自架伺服器部署模式（原因：深度整合 Claude Code CLI 執行環境，非獨立服務）
  • IDE 外掛（原因：Claude Code 已有 VS Code / JetBrains 擴充套件）
  • 非 Claude AI 模型（原因：深度整合 Claude Agent SDK subagent 機制）
  • GUI 介面（原因：CLI 是目標用戶的天然環境）
  • 即時協作編輯（原因：不在個人工具定位的核心使用情境中）

4.3 Future Scope（下一版本候選）

  • 多人協作 workflow / PR review 整合
  • 自訂 skill template 市場
  • 與 Linear / Jira ticket 自動同步
  • 支援更多 K8s 平台（AWS EKS / GKE）
  • 漸進式 pipeline 執行（基於 content hash 變更偵測）

6.2 Review Loop 流程

flowchart TD
    A["生成文件 / 程式碼"] --> B["subagent 以領域專家角色掃描"]
    B --> C{{"finding > 0？"}}
    C -->|"YES"| D["Claude 修復 CRITICAL + HIGH finding"]
    D --> E["verify subagent 重新掃描"]
    E --> F{{"finding 仍存在？"}}
    F -->|"YES < 5 次"| D
    F -->|"YES ≥ 5 次"| G["標記 [STUBBORN]，記入 TOTAL SUMMARY"]
    G --> H["git commit（per-round）"]
    F -->|"NO"| H
    C -->|"NO"| H
    H --> I{{"達停止條件？"}}
    I -->|"NO"| B
    I -->|"YES"| J["進入下一 STEP"]

6.3 Pipeline STEP 狀態機

stateDiagram-v2
    [*] --> PENDING: pipeline 啟動
    PENDING --> IN_PROGRESS: 開始執行 STEP
    IN_PROGRESS --> REVIEW_LOOP: STEP 有 ✅ 標記
    REVIEW_LOOP --> IN_PROGRESS: finding=0 且達停止條件
    IN_PROGRESS --> COMMITTED: git commit 成功
    COMMITTED --> PENDING: 進入下一 STEP
    IN_PROGRESS --> FAILED: 執行錯誤 / API 逾時
    FAILED --> IN_PROGRESS: /devsop-repair 修復
    COMMITTED --> [*]: 所有 31 STEP 完成

8. Constraints & Dependencies

8.1 技術限制

  • 必須使用 Claude Code CLI（核心 AI 執行環境，skill 機制依賴 Claude Code Agent SDK）
  • 必須有 Git（版本控制，每個 STEP 依賴 git commit 做狀態追蹤）
  • 必須有 Python 3（state file JSON 讀寫，Python AST 用於 Anti-Fake 稽核）

17. Privacy by Design & Data Protection

MYDEVSOP 為個人使用的 CLI 工具，不收集任何使用者個人資料（PII）。所有操作均在使用者本機執行。

#原則本產品實作方式合規狀態
1Proactive not Reactive設計階段即確保無 PII 收集已合規
2Privacy as Default無使用者帳號系統，無資料收集已合規
3Privacy Embedded into Design所有狀態（.devsop-state.json）存放在使用者本機已合規
5End-to-End SecuritySecrets 五層防護，不存入任何檔案已合規
6Visibility and Transparency開源，所有 skill 邏輯可讀已合規
7Respect for User Privacy使用者完全控制本機的所有數據已合規
例外：使用者使用 /devsop-idea 中的 WebSearch 功能時，搜尋關鍵字傳送給搜尋引擎 API，受搜尋引擎供應商的隱私政策約束。
